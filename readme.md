
Порівняння ефективності алгоритмів сортування
Опис завдання

Метою роботи було емпірично порівняти три алгоритми сортування:

Сортування вставками (Insertion Sort) — простий алгоритм зі складністю $O(N^2)$
Сортування злиттям (Merge Sort) — алгоритм "розділяй і володарюй" зі складністю $O(N \log N)$.
Timsort (стандартний sorted у Python) — гібридний алгоритм, що поєднує вставки та злиття.

Аналіз отриманих даних

1. Сортування вставками (Insertion Sort)
Малі дані (100): Працює швидко, майже миттєво.
Великі дані (10,000): Час виконання зростає катастрофічно. При збільшенні даних у 10 разів (з 1000 до 10000), час зріс приблизно у 100 разів. Це підтверджує теоретичну складність $O(N^2)$.
Висновок: Неефективний для великих масивів.

2. Сортування злиттям (Merge Sort)
Показує стабільний результат. Час зростає передбачувано і помірно (лінійно-логарифмічно).
На 10,000 елементах він у десятки разів швидший за Insertion Sort.
Недолік: У чистій реалізації на Python він програє Timsort через накладні витрати на рекурсію та створення нових списків.

3. Timsort (Вбудований)
Абсолютний лідер. Навіть на 10,000 елементах він працює за частки секунди.
Він швидший за "чистий" Merge Sort у 20-50 разів (у даному тесті).

Чому Timsort такий ефективний?
Timsort використовує найкращі сторони обох підходів:
Гібридність: Він розбиває масив на невеликі шматки (runs). Маленькі шматки він сортує Вставками (бо на малих даних Insertion Sort дуже швидкий і має мало "оверхеду").
Розумне злиття: Відсортовані шматки він об'єднує модифікованим Merge Sort.
Реалізація на C: Вбудована функція sorted написана на мові C, яка працює на низькому рівні значно швидше, ніж будь-який код, написаний на чистому Python.


Висновок: Саме тому програмісти використовують вбудовані методи sort() та sorted() — вони оптимізовані математично (алгоритмічно) та технічно (на рівні мови програмування).
